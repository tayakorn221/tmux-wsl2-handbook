01

Concept พื้นฐาน

WSL2 คืออะไร

WSL2 (Windows Subsystem for Linux v2) คือ Linux kernel จริง ที่รันใน lightweight virtual machine ของ Hyper-V บน Windows 11 ทำให้รัน distro อย่าง Ubuntu, Debian, Fedora ได้แบบ native ภายใน Windows โดยไม่ต้อง dual boot และไม่ต้องเปิด VirtualBox/VMware แยก

ประเด็นWSL1WSL2
สถาปัตยกรรมแปลง syscall Linux → Windows NT (translation layer)Linux kernel จริงใน Hyper-V VM
ความเข้ากันได้บาง syscall ไม่รองรับ (Docker daemon รันไม่ได้)รองรับ Linux เกือบ 100% — Docker, systemd ใช้ได้
Disk I/O (ใน Linux FS)ช้าเร็วกว่า WSL1 หลายเท่า
Disk I/O (ข้าม /mnt/c/)เร็วกว่า WSL2ช้ามาก (ผ่าน 9P protocol)
RAMใช้ตามจริงจองล่วงหน้า ปรับ cap ได้ใน .wslconfig
Networkใช้ stack ของ Windows ตรง ๆvirtual NIC แยก + port forwarding อัตโนมัติ
สรุปWSL2 เป็นค่า default ปัจจุบันและเป็นตัวเลือกที่ถูกต้องสำหรับ dev workflow ทั้งหมด ยกเว้นกรณีพิเศษที่ต้องการ I/O cross-OS เร็ว ๆ (เกือบไม่มีในงานจริง)

tmux คืออะไร

tmux (terminal multiplexer) คือโปรแกรมที่ให้รัน terminal session หลายอันใน window เดียว และที่สำคัญที่สุดคือ session ทำงานต่อใน background ได้แม้ปิด terminal ไปแล้ว — กลับมา tmux attach ก็ได้ทุก process ที่รันอยู่กลับคืน

tmux ต่างจาก Windows Terminal tab อย่างไร

ประเด็นWindows Terminal tab/panetmux
Persistenceปิด terminal = process ตายหมดdetach แล้ว process ยังรันใน background
Remote/SSHtab อยู่ฝั่ง Windows เท่านั้นsession อยู่ฝั่ง server — SSH ขาดแล้ว reconnect ทำงานต่อได้
Layout templatesทำไม่ได้scriptable ผ่าน shell หรือ tmuxinator/tmuxp
Keybindingจำกัดrebind ได้ทั้งหมด
Cross-platformWindows onlyทุก Unix-like ใช้ config เดียวกันได้

ใช้คู่กัน: Windows Terminal เป็น "หน้าต่าง" → เปิด WSL2 → ใน WSL2 รัน tmux เพื่อจัดการ session/window/pane

Mental Model 4 ชั้น

นี่คือแบบจำลองที่ทำให้ทุกอย่างที่เหลือเข้าใจง่าย — tmux ซ้อนกัน 4 ชั้นจากนอกสุดเข้าใน:

ชั้นทำหน้าที่อะไรanalogy
serverdaemon ดูแลทุก session — เกิดเมื่อสั่ง tmux ครั้งแรก ดับเมื่อทุก session หายบริการระบบ
sessionหน่วยใหญ่สุด — ตั้งชื่อได้ (kmutt, nextjs) detach/attach อิสระ"โปรเจกต์"
windowเหมือน tab — มีหมายเลข ตั้งชื่อได้ แต่ละ window มี layout pane ของตัวเอง"tab"
paneterminal จริง — รัน shell หนึ่งตัว split window เป็นหลายส่วน"split"

ทำไมต้องใช้ tmux + WSL2 คู่กัน

ปัญหาที่แก้tmux ช่วยอะไร
Windows Terminal crash / restart เครื่องsession ใน tmux ยังอยู่ (ตราบใดที่ WSL2 ยังไม่ shutdown)
ต้องเปิดหลาย tab สำหรับโปรเจกต์เดียวจัด pane ใน window เดียว เห็นทุกอย่างพร้อมกัน
ลืมว่าทำอะไรไว้เมื่อวานattach กลับ ได้ทุก process + history + layout เดิม
Long-running job (dbt run, npm build)detach ทิ้งไว้ — ไม่ต้องคา terminal เปิดทั้งวัน
ต้องโชว์งาน 2 หน้าจอwindow สำหรับ code, อีก window สำหรับ logs/db — สลับด้วย keystroke

02

การติดตั้ง

2.1 เช็คก่อนว่ามี WSL2 อยู่หรือยัง

เปิด PowerShell (ไม่ต้อง admin):

wsl -l -v

ผลลัพธ์ที่เป็นไปได้:

  • ขึ้น list distro พร้อม VERSION 2 → มี WSL2 แล้ว ข้ามไป 2.3
  • ขึ้น list แต่ VERSION 1 → ดู 2.5 (migrate)
  • error ...has no installed distributions → ทำ 2.2
  • error เกี่ยวกับ feature → ดู 2.4 (troubleshoot virtualization)

2.2 ติดตั้ง WSL2 + Ubuntu

เปิด PowerShell แบบ Run as Administrator แล้ว:

wsl --install

คำสั่งเดียวนี้จะ (อัตโนมัติ): เปิด feature VirtualMachinePlatform + Microsoft-Windows-Subsystem-Linux · ตั้ง default WSL version = 2 · download + install Ubuntu · ขอ reboot

หลัง reboot, Ubuntu จะเปิดเองและขอตั้ง username/password (UNIX account ไม่ใช่ Microsoft account) ถ้าอยากเลือก distro อื่น:

wsl --list --online                # ดู distro ที่ install ได้
wsl --install -d Ubuntu-24.04

2.3 ตั้ง default version + update

wsl --set-default-version 2
wsl --update
wsl -l -v

ควรเห็น STATE = Running/Stopped และ VERSION = 2

2.4 Troubleshoot: virtualization ไม่เปิดใน BIOS

ถ้า wsl --install แจ้ง error เกี่ยวกับ Hyper-V หรือ virtualization:

  1. เช็ค Task Manager → Performance → CPU → บรรทัด "Virtualization" ต้องเป็น Enabled
  2. ถ้า Disabled: รีสตาร์ทเข้า BIOS/UEFI (กด F2/Del/F10 ตอนบูต)
  3. หาเมนู Intel VT-x / AMD-V / SVM Mode → enable → save → exit
  4. กลับ Windows รัน wsl --install ใหม่

ถ้ายังไม่ได้ เปิด feature ด้วยมือ (PowerShell as admin):

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

แล้ว reboot

2.5 Migrate WSL1 → WSL2

wsl --set-version Ubuntu 2

ใช้เวลา 5–30 นาทีขึ้นกับขนาด disk image รอจน Conversion complete. แล้วเช็ค wsl -l -v

2.6 ลง tmux ใน Ubuntu

$ sudo apt update
$ sudo apt install -y tmux
$ tmux -V

ควรเห็น tmux 3.x.x ขึ้นไป — Ubuntu 22.04+ ได้ 3.2a เป็น default ซึ่งพอใช้ feature ในคู่มือนี้

2.7 เริ่มใช้ครั้งแรก

$ tmux                          # เปิด session ไม่ตั้งชื่อ
$ tmux new -s test              # เปิด session ชื่อ "test"

ถ้าเห็นแถบสถานะสีเขียวด้านล่าง = ทำงานแล้ว ออกด้วย Ctrl+d หรือพิมพ์ exit

tmux Basics

3.1 Prefix Key Concept

ทุก keybinding ใน tmux ต้องกด prefix key ก่อน แล้วตามด้วย key สั่ง คล้าย leader key ใน Vim หรือ Ctrl+x ใน Emacs

Prefixที่มาข้อดีข้อเสีย
Ctrl+bdefault ของ tmuxไม่ชน Emacs conventionกดยาก (สองมือ), ไกลจากเหย้า
Ctrl+aจาก GNU Screenกดง่าย, ergonomicชนกับ bash "ไปต้นบรรทัด"
Why Ctrl+aกดด้วยมือซ้ายเดียว (นิ้วก้อย + นิ้วชี้) ลดอาการล้า keystroke ที่ใช้บ่อยที่สุดหลังลง tmux — ตลอดคู่มือนี้ prefix = Ctrl+a และ prefix + X หมายถึงกด Ctrl+a ปล่อย แล้วกด X

Configuration

ไฟล์ tmux.conf ที่แนบใน repo มี comment อธิบายทุกบรรทัด

4.1 ติดตั้ง config

# จาก WSL2 (สมมติ clone repo นี้มาที่ ~/tmux-wsl2-handbook/)
$ cp ~/tmux-wsl2-handbook/tmux.conf ~/.tmux.conf

# หรือ symlink (อัพเดต config ผ่าน git pull ได้ทันที)
$ ln -s ~/tmux-wsl2-handbook/tmux.conf ~/.tmux.conf

4.2 Reload config

ถ้าอยู่ใน tmux session อยู่แล้ว ไม่ต้องปิด ใช้ keybinding ที่ตั้งไว้: prefix + r → จะ source ~/.tmux.conf ใหม่ + แสดง tmux.conf reloaded ใน status bar

ถ้าอยู่นอก tmux (หรือ binding ยังไม่ทำงานเพราะเพิ่ง install):

$ tmux source-file ~/.tmux.conf

05

Workflow Templates

ทุก template สร้าง session พร้อม pane layout ครบในคำสั่งเดียว ใช้ shell command ปกติ (ไม่ต้องลง plugin) — ก๊อปวางใน WSL2 ได้เลย

5.1 Template: KMUTT DataPlatform session

เป้าหมาย: 1 session, 2 window — Window 1 dbt: 3 pane (Claude Code ซ้ายเต็มความสูง · dbt ขวาบน · psql ขวาล่าง) · Window 2 logs: ดู docker logs

# ปรับ path ตามจริง
PROJECT_DIR="$HOME/projects/kmutt-dataplatform"
PG_USER="postgres"; PG_DB="dataplatform"

# สร้าง session + window แรก ที่ pane เริ่มต้นรัน claude
tmux new-session -d -s kmutt -n dbt -c "$PROJECT_DIR"
tmux send-keys -t kmutt:dbt 'claude' C-m

# split ขวา (pane 2) สำหรับ dbt
tmux split-window -h -t kmutt:dbt -c "$PROJECT_DIR"
tmux send-keys -t kmutt:dbt.2 'source .venv/bin/activate && dbt debug' C-m

# split ขวาล่าง (pane 3) สำหรับ psql
tmux split-window -v -t kmutt:dbt.2 -c "$PROJECT_DIR"
tmux send-keys -t kmutt:dbt.3 "psql -h localhost -U $PG_USER -d $PG_DB" C-m

# window 2 — logs
tmux new-window -t kmutt -n logs -c "$PROJECT_DIR"
tmux send-keys -t kmutt:logs 'docker compose logs -f' C-m

# focus กลับ window แรก pane Claude แล้ว attach
tmux select-window -t kmutt:dbt; tmux select-pane -t kmutt:dbt.1
tmux attach -t kmutt

เก็บเป็น script ~/bin/kmutt-up.sh แล้ว chmod +x — เรียกครั้งเดียวจบ

5.2 Template: Next.js Freelance Project

เป้าหมาย: Window 1 dev: 4 pane (Claude / npm run dev / git / db client) · Window 2 prisma: prisma studio

PROJECT_DIR="$HOME/projects/freelance-app"
tmux new-session -d -s nextjs -n dev -c "$PROJECT_DIR"
tmux send-keys -t nextjs:dev 'claude' C-m                 # pane 1 ซ้ายบน
tmux split-window -h -t nextjs:dev -c "$PROJECT_DIR"
tmux send-keys -t nextjs:dev.2 'npm run dev' C-m          # pane 2 ขวาบน
tmux split-window -v -t nextjs:dev.1 -c "$PROJECT_DIR"
tmux send-keys -t nextjs:dev.3 'git status' C-m          # pane 3 ซ้ายล่าง
tmux split-window -v -t nextjs:dev.2 -c "$PROJECT_DIR"
tmux send-keys -t nextjs:dev.4 'pgcli postgres://app:app@localhost:5432/app' C-m
tmux new-window -t nextjs -n prisma -c "$PROJECT_DIR"
tmux send-keys -t nextjs:prisma 'npx prisma studio' C-m
tmux select-window -t nextjs:dev; tmux select-pane -t nextjs:dev.1
tmux attach -t nextjs

หลัง npm run dev ขึ้นมาแล้ว เปิด Chrome บน Windows ที่ http://localhost:3000 — WSL2 forward port ให้อัตโนมัติ (ดู 6.4)

5.3 Template: Daily Standup Workflow

ใช้ tmux ให้ครบ cycle ของวัน — เลิกงาน detach, วันถัดไป attach กลับ

ช่วงเวลาทำอะไร
เช้า (วันแรก)~/bin/kmutt-up.sh หรือ ~/bin/nextjs-up.sh
ไปทานข้าว/ประชุมprefix + d (detach — ทุกอย่างยังรัน) ปิด Windows Terminal/ฝา laptop ได้
กลับมาทำต่อtmux ls → tmux a -t kmutt
เลิกงานprefix + d · อย่ารัน wsl --shutdown
สิ้นสัปดาห์ cleantmux kill-session -t nextjs หรือ tmux kill-server
Why ไม่ใช้แค่ Windows Terminal tabtab ของ Windows Terminal ผูกกับ process Windows — ปิดหน้าต่าง = ทุก process ใน WSL ตาย แต่ tmux session อยู่ฝั่ง Linux ตราบใดที่ WSL2 ไม่ shutdown session ยังอยู่ครบ

06

WSL2 ↔ Windows Integration

6.1 Filesystem: /home/ vs /mnt/c/

ถ้าโปรเจกต์เดิมอยู่ใน /mnt/c/Users/.../ ย้ายเข้า Linux FS:

$ mkdir -p ~/projects
$ mv /mnt/c/Users/STUDENT/Documents/myproject ~/projects/

6.2 เข้าถึงไฟล์ WSL จาก Windows

ใน File Explorer พิมพ์ใน address bar: \\wsl$\Ubuntu\home\<user>\ หรือบน Windows 11: \\wsl.localhost\Ubuntu\home\<user>\ · ทางลัดจาก WSL terminal:

$ explorer.exe .          # เปิด Explorer ที่ folder ปัจจุบัน

6.3 VS Code Remote-WSL

Install (ครั้งเดียว): ลง VS Code บน Windows → ติดตั้ง extension "WSL" (Microsoft) → restart · ใช้งาน จาก WSL terminal:

$ cd ~/projects/kmutt-dataplatform
$ code .

ครั้งแรก VS Code จะ install vscode-server ใน WSL อัตโนมัติ — extension (Python, Node, debugger) รันฝั่ง Linux ส่วน UI ฝั่ง Windows · เช็คมุมล่างซ้ายขึ้น WSL: Ubuntu

6.4 Network: localhost Port Forwarding

Windows 11 + WSL2 รุ่นใหม่มี localhost relay อัตโนมัติ — เปิด server ใน WSL ที่ 0.0.0.0:3000 แล้วเปิด Chrome ฝั่ง Windows ที่ http://localhost:3000 เห็นทันที

ถ้าใช้ไม่ได้ (ดู 8.5)เช็คว่า server bind 0.0.0.0 ไม่ใช่ 127.0.0.1 (บาง framework default 127.0.0.1) · รีสตาร์ท wsl --shutdown · ปิด Windows Defender Firewall ทดสอบช่วงสั้น ๆ

6.5 เปิด Windows app จาก WSL

คำสั่งใน WSLทำอะไร
explorer.exe .เปิด File Explorer ที่ folder ปัจจุบัน
code .เปิด VS Code Remote-WSL
clip.exe < file.txtcopy เนื้อหาไฟล์เข้า Windows clipboard
cmd.exe /c start https://...เปิด URL ด้วย default browser
wslview <file>เปิดด้วย default Windows app (เหมือน xdg-open)

Troubleshooting

8.1 tmux session ค้าง, ต้อง kill ทุก session

$ tmux kill-server
# ถ้า tmux ls / tmux a ค้าง:
$ pkill -9 tmux                            # ฆ่า process ทั้งหมด
$ rm -rf /tmp/tmux-$(id -u)                # ลบ socket directory
$ tmux ls                                  # ควรเห็น "no server running"

8.2 Config ไม่ reload หลังแก้

# 1. ทดสอบ syntax (ถ้า error จะ print ออกมา)
$ tmux source-file ~/.tmux.conf
# 2. เช็คว่า binding ถูก register หรือยัง
$ tmux list-keys | grep -i "source-file"
# 3. อยู่ใน session ที่ start ก่อนเพิ่ม binding? → ใช้คำสั่งใน 1 หรือ kill-server แล้วเปิดใหม่

8.3 WSL2 กิน RAM เยอะเกินไป

WSL2 จอง RAM แบบ dynamic แต่ไม่คืนให้ Windows ทันที — สร้าง C:\Users\<WindowsUser>\.wslconfig:

[wsl2]
memory=8GB              # cap RAM (เครื่อง 16GB ใช้ 8GB, 32GB ใช้ 16GB)
processors=4            # cap จำนวน vCPU
swap=4GB
localhostForwarding=true

แล้วบังคับ apply: wsl --shutdown → เปิด WSL ใหม่ · .wslconfig มีผลทุก distro ภายใต้ WSL2

8.4 ใช้ wsl --shutdown ตอนไหน

ใช้เมื่อ ✓ห้ามใช้เมื่อ ✗
เพิ่งแก้ .wslconfig ต้องการ applyมี tmux session ที่ยังไม่อยากให้ตาย (จะถูก SIGKILL ทันที)
RAM ของ WSL2 บวม ต้องการคืนให้ Windowsกำลังมี long-running job (dbt seed, npm build) → งานพัง
WSL2 hang / network เพี้ยน

8.5 Port ใน WSL ไม่ถึง Windows browser

# 1. Server bind ที่อะไร?
$ ss -tlnp | grep 3000
#    ต้องเห็น 0.0.0.0:3000 หรือ *:3000 — ถ้า 127.0.0.1:3000 ให้แก้ framework ให้ bind 0.0.0.0

# 2. Firewall block? (PowerShell as admin, ทดสอบชั่วคราว)
New-NetFirewallRule -DisplayName "WSL2 dev" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 3000

# 3. รีสตาร์ท WSL
wsl --shutdown

# 4. Windows 11 บางเวอร์ชันมี bug — เพิ่ม manual port proxy (admin)
$wslIp = (wsl hostname -I).Trim().Split()[0]
netsh interface portproxy add v4tov4 listenport=3000 listenaddress=0.0.0.0 connectport=3000 connectaddress=$wslIp

8.6 Clock skew หลัง laptop sleep

อาการ: หลัง wake from sleep ใช้ git, npm, apt, JWT auth พัง เพราะเวลาใน WSL2 ห่างจาก Windows หลายชั่วโมง

# แก้ทันที
$ sudo hwclock -s          # อ่านเวลาจาก hardware clock (= Windows)

# แก้ถาวร — systemd-timesyncd (Ubuntu 22.04+ มีให้แล้ว)
$ sudo apt install -y systemd-timesyncd
$ sudo systemctl enable --now systemd-timesyncd
$ timedatectl status

09

ขั้นต่อไป (Advanced)

เมื่อใช้ tmux จนคล่อง ค่อยลองเพิ่ม:

9.1 tpm (tmux Plugin Manager)

จัดการ plugin tmux ด้วย git ผ่าน 1 บรรทัดใน .tmux.conf

$ git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm

เพิ่มท้าย ~/.tmux.conf:

set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible'
run '~/.tmux/plugins/tpm/tpm'

Install plugin: prefix + I (capital i)

9.2 tmux-resurrect + tmux-continuum

ปัญหา: wsl --shutdown แล้ว session หายหมด ต้อง bootstrap ใหม่ · แก้: 2 plugin นี้ save/restore session อัตโนมัติทุก 15 นาที — เปิด tmux แล้วทุก window/pane กลับมาเลย

set -g @plugin 'tmux-plugins/tmux-resurrect'
set -g @plugin 'tmux-plugins/tmux-continuum'
set -g @continuum-restore 'on'

prefix + Ctrl+s save, prefix + Ctrl+r restore (manual)

9.3 tmuxinator / tmuxp

แทนที่ shell script ใน Section 5 ด้วย YAML declarative — config อ่านง่ายและ portable · tmuxinator (Ruby): mux kmutt · tmuxp (Python): tmuxp load nextjs

แนะนำ tmuxpถ้าคุณใช้ Python ใน data stack อยู่แล้ว — ไม่ต้องลง Ruby เพิ่ม
# ~/.tmuxp/kmutt.yaml
session_name: kmutt
start_directory: ~/projects/kmutt-dataplatform
windows:
  - window_name: dbt
    layout: main-vertical
    panes:
      - claude
      - source .venv/bin/activate && dbt debug
      - psql -h localhost -U postgres -d dataplatform
  - window_name: logs
    panes:
      - docker compose logs -f

9.4 Integration กับ Neovim

ใน init.lua เพิ่ม plugin christoomey/vim-tmux-navigator — กด Ctrl+h/j/k/l จะ navigate ข้าม Neovim split ↔ tmux pane ได้แบบไร้รอยต่อ (ไม่ต้องกด prefix) · ใน .tmux.conf ต้องเพิ่ม smart-detect (ดูเอกสาร plugin) — power-user setup ที่คุ้มสุด ๆ ถ้าใช้ Neovim เป็นหลัก
